# Тестовое задание

ФИ: Голубева Ульяна

## Описание проекта

Веб-приложение для отображения информации обо всех сотрудниках компании в виде списка с возможностью фильтрации по департаментам, поиска, сортировки и просмотра детальной информации. Данные загружаются с предоставленного API.

## План реализации

### Шаг 1. Инициализация проекта

1. Создание чистого vite проекта на основе шаблона react-ts;
2. Установка зависимостей(React Router, Axios, Redux Toolkit, styled-components);
3. Настройка базовой структуры проекта.

### Шаг 2. Проектирование структуры проекта

Создание папок/примерных файлов

а. app(роутинг);
b. components;
  ~ UI;
  ~ все компоненты на основе UI;
c. pages(страницы, собранные из компонентов);
d. services(работа с API);
e. store(конфигурация стора и слайсы);
f. types;
g. hooks(обертки над useSelector, useDispatch);
h. utils;
i. App.tsx;
j. main.tsx;
k. index.css.

### Шаг 3. Реализация UI (без логики)

На этом этапе буду верстать экраны с моковыми данными, чтобы проработать UX и дизайн:

1. Экран загрузки (Loader).
2. Экран критической ошибки (Error).
3. Главный экран со списком пользователей (HomePageUI).
4. Пустой результат поиска (Empty).
5. Экран деталей пользователя (UserPageUI).

На этом этапе не подключаю API и Redux — только визуальная часть.

### Шаг 4. Работа с API и Redux

Реализовать сервис для работы с API:
1. Получение списка всех пользователей;
2. Получение пользователей по департаменту.

Конфигурация стора, создание слайса:
1. store/index.ts;
2. userSlice.ts с состоянием(предположительно):
  a. users;
  b. isLoading;
  c. error.

### Шаг 5. Создание компонентов (бизнес-логика)

На основе UI создаются компоненты с логикой:

1. Подключаем стор и сервисы.
2. Компоненты берут данные из стора и диспатчат экшены.

Примеры:

1. UserCard — использует UserCardUI + данные из стора;
2. UserList — отображает список пользователей;
3. TopAppBar — фильтры и поиск;
4. SortModal — сортировка.

### Шаг 6. Создание страниц

На основе компонентов создаются страницы приложения, которые управляют состоянием и роутингом:

1. Главная страница (HomePage) - список пользователей, фильтры, поиск, сортировка;
2. Страница деталей (UserPage) - отображение выбранного пользователя;
3. Страница ошибок (ErrorPage) - отображается при проблемах с API;
4. Экран загрузки (LoaderPage) - пока идёт загрузка данных.

Особенности:

1. Страницы используют компоненты с бизнес-логикой;
2. Диспатчат запросы к API.
